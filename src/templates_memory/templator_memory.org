# TREXIO in-memory (RAM) back end
#
# Same-process, disk-free back end: all data lives in the per-group structs
# held by the open handle. It mirrors the TEXT back end but with no file I/O:
# reading a group returns the in-RAM struct (an empty one if not yet present),
# and flushing is a no-op. The struct-based categories (numerical attributes,
# numerical/string datasets, string attributes) are supported; sparse,
# buffered and determinant data are not yet implemented for this back end and
# the front end returns TREXIO_NOT_IMPLEMENTED for them.
